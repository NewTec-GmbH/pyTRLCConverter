   .. py:method:: _apply_table_options(html_text: str, table_options: dict) -> str
      :staticmethod:


      Apply table rendering options to generated HTML.

      Replaces the plain ``<table>`` tag with one carrying a ``style`` attribute when
      ``"border"`` is set, and injects a ``style`` attribute into every ``<th>`` tag
      when ``"headingStyle"`` is set.  Values are HTML-escaped before insertion.

      :param html_text: HTML string produced by the Markdown renderer.
      :type html_text: str
      :param table_options: Table options with optional keys ``"border"`` (str) and
                            ``"headingStyle"`` (str), both interpreted as CSS style values.
      :type table_options: dict

      :returns: HTML string with table styling applied.
      :rtype: str

   .. py:method:: _render(package_name: str, type_name: str, attribute_name: str, attribute_value: str) -> str

      Render an attribute value to an XHTML-wrapped string based on the render configuration.

      If the attribute is configured as CommonMark Markdown, it is converted via marko.
      If configured as GFM, it is converted with the GFM extension.
      If configured as XHTML, the value is passed through as-is inside the XHTML wrapper.
      If configured as path, the value is treated as a file path and converted to an XHTML
      ``<object>`` element; the file is scheduled for copying to the output folder.
      Otherwise plain text is HTML-escaped and wrapped.

      For Markdown and GFM formats, optional table rendering options (``border``,
      ``headingStyle``) from the render configuration are applied to the generated HTML.

      :param package_name: TRLC package name of the record.
      :type package_name: str
      :param type_name: TRLC type name of the record.
      :type type_name: str
      :param attribute_name: Attribute field name.
      :type attribute_name: str
      :param attribute_value: Raw attribute value string.
      :type attribute_value: str

      :returns: XHTML-wrapped content string.
      :rtype: str

   .. py:method:: convert_section(section: str, level: int) -> pyTRLCConverter.ret.Ret

      Process the given section item as a named SPEC-HIERARCHY container.

      The section reuses the last processed TRLC record's SPEC-OBJECT as the mandatory
      SPEC-OBJECT-REF in the SPEC-HIERARCHY node (the ReqIF v1.2 schema requires
      minOccurs=1). The section title is carried by the hierarchy node's LONG-NAME.
      No dedicated section SPEC-OBJECT is created.

      If no record has been processed yet, the section name is used as the document
      title (specification long-name) for the very first such section.  Any further
      section without a preceding record emits a warning and is skipped.

      :param section: The section name
      :type section: str
      :param level: The section indentation level
      :type level: int

      :returns: Status
      :rtype: Ret
